**Goal:** On the central's Sites page, move **Enroll site** into the ⋯ menu and add **Download central certificate**; serve the cert from an authed api endpoint; and offer the same download in the enroll-success dialog — so a remote lab operator gets the cert it needs to trust the central.

**Architecture:** New authed endpoint `GET /api/settings/sync/central-certificate` reads a PEM file at `TLS_CERT_PATH` (installer-mounted `fullchain.pem`) and streams it as a download. Studio `api.ts` gains `downloadCentralCertificate()`; `Sites.tsx` header ⋯ menu gains Enroll + Download, and the reveal dialog gains a Download action. Installer mounts the central's cert into the api and sets `TLS_CERT_PATH`.

**Tech Stack:** TypeScript, Zod (`@openldr/config`), Fastify (`apps/server`), Vitest, React/shadcn (`apps/studio`), docker-compose + install scripts.

**Spec:** `docs/superpowers/specs/2026-07-19-remote-central-enrollment-ux-design.md` (Slice 2). **Branch:** `claude/remote-central-ux-s2`, worktree `D:\Projects\openldr-remote-ux`.

**Facts:** routes live in `apps/server/src/settings-routes.ts` (`app.get('/api/settings/sync/...', { preHandler: requireRole('lab_admin') }, ...)`; ⚠ this file registers under global compression — a raw `reply.send(x)` in an async handler can clobber the body, so **`return reply.send(...)`**). `ctx.cfg` holds parsed config. - [ ] **Step 3: Implement the route.** In `settings-routes.ts` add near the other `/api/settings/sync/*` GETs. Import `readFile` at top: `import { readFile } from 'node:fs/promises';`. Handler:
```ts
  // Serve this server's public TLS cert (PEM) so a remote lab can trust a self-signed central.
  // The cert is public (presented in every TLS handshake) but the route is lab_admin like the rest
  // of /api/settings/*. Path is TLS_CERT_PATH (installer-mounted fullchain.pem).
  app.get('/api/settings/sync/central-certificate', { preHandler: requireRole('lab_admin') }, async (_req, reply) => {
    const path = ctx.cfg.TLS_CERT_PATH;
    if (!path) throw appError('SY0404', { message: 'no TLS certificate is configured (set TLS_CERT_PATH / mount the cert)' });
    let pem: string;
    try { pem = await readFile(path, 'utf8'); }
    catch { throw appError('SY0404', { message: 'configured TLS certificate file was not found' }); }
    reply.header('content-type', 'application/x-pem-file');
    reply.header('content-disposition', 'attachment; filename="central-certificate.pem"');
    return reply.send(pem); // `return` is load-bearing under global compression (see file header note)
  });
```
Use the file's existing coded-error helper (`appError`/`throw appError('SY0404', …)`) — check how the enroll route or others in this file raise coded errors and match it. If the file maps errors differently, follow its convention so the 404 carries the app's `{ error, code, correlationId }` body.

- [ ] **Step 1: api.ts — `downloadCentralCertificate()`** (mirror `downloadScheduleRun`):
```ts
export async function downloadCentralCertificate(): Promise<void> {
  const res = await authFetch('/api/settings/sync/central-certificate');
  if (!res.ok) throw Object.assign(new Error('cert download failed'), { status: res.status });
  const blob = await res.blob();
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url; a.download = 'central-certificate.pem';
  document.body.appendChild(a); a.click(); a.remove();
  URL.revokeObjectURL(url);
}
```
