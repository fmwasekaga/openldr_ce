# i18n SP-A — UI Internationalization (en/fr/pt) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The `apps/web` UI fully renders in English, French, and Portuguese with a user-selectable, persisted language.

**Architecture:** Split the single `i18n/index.ts` `en` object into per-language files (`en.ts`/`fr.ts`/`pt.ts`); `fr`/`pt` are typed `: typeof en` so the build fails on any key drift. Extract remaining hardcoded UI strings (nav labels, aria/title) into keys. A `language.ts` helper persists/restores the choice in `localStorage`; a switcher in the `AppShell` user dropdown changes the language.

**Tech Stack:** React, react-i18next, i18next, Vitest + Testing Library, Turborepo/pnpm. Spec: `docs/superpowers/specs/2026-06-22-i18n-sp-a-ui-design.md`.

**Conventions:**
- Web tests from root: `pnpm --filter @openldr/web test -- --run <path>`. Typecheck: `pnpm --filter @openldr/web exec tsc -p tsconfig.json --noEmit`.
- Full gate (final task): `pnpm turbo typecheck lint test build && pnpm depcruise`.
- `@openldr/web#test` can flake under turbo parallelism — re-run in isolation.
- Commit after every task.
- **Translation rules (apply in Tasks 4–5):** translate every value; preserve interpolation placeholders verbatim (`{{error}}`, `{{id}}`, `{{username}}`, `{{name}}`, `{{when}}`, `{{facility}}`, etc.); keep HTML/markup if any; append a trailing `// review` line-comment on any clinical/AMR/technical term a native speaker should verify (antibiogram, first-isolate, specimen, specimen-origin, resourceType names, ValueSet/CodeSystem/ConceptMap, DHIS2, orgUnit, etc.); do NOT translate proper nouns/product names (OpenLDR, DHIS2, LOINC, SNOMED, WHONET, FHIR, Keycloak) or `data-testid`s.

```ts
import { describe, it, expect } from 'vitest';
import { en } from './en';

/** Recursively collect dotted key paths from a nested resource object. */
function keyPaths(obj: Record<string, unknown>, prefix = ''): string[] {
  return Object.entries(obj).flatMap(([k, v]) => {
    const path = prefix ? `${prefix}.${k}` : k;
    return v && typeof v === 'object' && !Array.isArray(v) ? keyPaths(v as Record<string, unknown>, path) : [path];
  });
}

const enKeys = keyPaths(en).sort();

describe('i18n key parity', () => {
  it('en has a non-trivial key set', () => {
    expect(enKeys.length).toBeGreaterThan(100);
  });
  // fr/pt assertions are added in Tasks 4 and 5 once those bundles exist:
  //   it('fr matches en', () => expect(keyPaths(fr).sort()).toEqual(enKeys));

});
```

- [ ] **Step 3: Implement** — `language.ts`

```ts
import i18n from './index';

export const STORAGE_KEY = 'openldr.lang';
export const SUPPORTED_LANGUAGES = [
  { code: 'en', label: 'English' },
  { code: 'fr', label: 'Français' },
  { code: 'pt', label: 'Português' },
] as const;
export type LanguageCode = (typeof SUPPORTED_LANGUAGES)[number]['code'];

function isSupported(v: string | null): v is LanguageCode {
  return v === 'en' || v === 'fr' || v === 'pt';
}

export function getStoredLanguage(): LanguageCode {
  try {
    const v = localStorage.getItem(STORAGE_KEY);
    return isSupported(v) ? v : 'en';
  } catch { return 'en'; }
}

export async function setLanguage(code: LanguageCode): Promise<void> {
  try { localStorage.setItem(STORAGE_KEY, code); } catch { /* ignore */ }
  await i18n.changeLanguage(code);
}
```

- [ ] **Step 4: Restore on init** — in `index.ts`, after the `init(...)` call:
