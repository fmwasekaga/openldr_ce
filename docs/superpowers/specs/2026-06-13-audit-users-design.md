# Audit Log + Decoupled Users — Design Spec

**PRD mapping:** §8 build-sequence step 7 — P1-AUD-1 (append-only audit log: actor/action/entity/before-after/timestamp), P1-AUD-2 (audit integrates with provenance so ingest events are accountable alongside user actions), P1-USER-1 (user management decoupled from Keycloak, reimplementing the Corlix pattern, so IdPs swap behind the OIDC port). CLI `audit`/`user` commands (P1-CLI-1/2).

**Status:** Approved design (2026-06-13). One combined sub-project (audit + users). Backend only — no UI this round (the dashboard's Forms/Users/Audit nav entries stay disabled stubs until a later UI pass).

---

## 1. Key decisions (locked during brainstorming)

1. **Combined sub-project** — `@openldr/audit` + `@openldr/users` in one spec→plan→build cycle (related: audit records the actor the users module identifies; both internal-DB tables).
2. **Audit coverage = operational events now + open API.** Ship the store + `recordAuditEvent` + query/CLI, AND instrument what actually mutates today: ingest batch done/failed (system actor → P1-AUD-2), plugin install/remove, db reset. User-action auditing slots in for free when auth + CRUD land.
3. **Users = CLI-managed now + JIT-ready `syncFromClaims`.** Local `users` table keyed by an internal id with a nullable `subject` (OIDC `sub`); CLI manages users now; `syncFromClaims` is built + unit-tested, ready for auth to call on login. Decoupled from the IdP.

## 2. Topology

Both packages fill existing placeholders and import only `@openldr/db` (`InternalSchema` + Kysely) and `@openldr/core` (`Logger`, `errorMessage`) — **no `adapter-*`** (DP-1). Bootstrap (the composition root) exposes `ctx.audit` + `ctx.users` over the internal Kysely and wires the operational audit hooks. CLI adds `audit` + `user` command groups. No new app/UI surface.

```
CLI / ingest worker / plugin admin / db reset
        │ record(...)                 │ list / create / set-role / syncFromClaims
   @openldr/audit (AuditStore)   @openldr/users (UserStore)
        └────────── Kysely<InternalSchema> (internal Postgres) ──────────┘
```

## 3. Schema (internal migrations)

### 3.1 `audit_events` (migration 005) — append-only
| column | type | notes |
|--------|------|-------|
| `id` | uuid (text) pk | generated by the caller (`randomUUID`) |
| `occurred_at` | timestamptz default now | |
| `actor_type` | text | `'user'` \| `'system'` |
| `actor_id` | text null | user id / subject; null for system |
| `actor_name` | text | display label (`'system'`, username) |
| `action` | text | dotted verb, e.g. `ingest.batch.done`, `plugin.install`, `db.reset`, `user.create` |
| `entity_type` | text | `batch` \| `plugin` \| `database` \| `user` … |
| `entity_id` | text | the affected entity's id |
| `before` | jsonb null | prior value where relevant |
| `after` | jsonb null | new value where relevant |
| `metadata` | jsonb null | extra context (provenance, counts, error) |

Indexes: `occurred_at`, `(entity_type, entity_id)`, `actor_id`. **Append-only** — the store exposes no update/delete.

### 3.2 `users` (migration 006)
| column | type | notes |
|--------|------|-------|
| `id` | uuid (text) pk | internal identity (stable across IdP changes) |
| `subject` | text unique null | the OIDC `sub`; null until first login links it |
| `username` | text unique not null | login/display handle |
| `display_name` | text null | |
| `email` | text null | |
| `roles` | jsonb (`string[]`) default `[]` | free-form strings in P1 (no enforcement yet) |
| `status` | text default `'active'` | `'active'` \| `'disabled'` |
| `created_at` / `updated_at` | timestamptz default now | |
| `last_login_at` | timestamptz null | bumped by `syncFromClaims` |

The nullable `subject` + unique `username` is the decoupling seam: CLI pre-creates a user (subject null); `syncFromClaims` links the IdP `sub` on first login by matching username. Local identity owns profile/roles; the IdP owns authn.

## 4. `@openldr/audit`

```ts
export interface AuditEventInput {
  actorType: 'user' | 'system';
  actorId?: string | null;
  actorName: string;
  action: string;
  entityType: string;
  entityId: string;
  before?: unknown;
  after?: unknown;
  metadata?: Record<string, unknown>;
}
export interface AuditEvent extends AuditEventInput { id: string; occurredAt: string }
export interface AuditFilter { actorId?: string; entityType?: string; entityId?: string; action?: string; from?: string; to?: string; limit?: number }

export interface AuditStore {
  record(e: AuditEventInput): Promise<AuditEvent>;   // insert only (append-only)
  list(filter?: AuditFilter): Promise<AuditEvent[]>; // newest first, default limit 100
  get(id: string): Promise<AuditEvent | undefined>;
}
export function createAuditStore(db: Kysely<InternalSchema>): AuditStore;

/** Best-effort recorder for operational hooks: never throws into the caller. */
export function safeRecord(store: AuditStore, logger: Logger, e: AuditEventInput): Promise<void>;
```

`record` stamps `id = randomUUID()` and `occurred_at = now()` (DB default). `safeRecord` wraps `record` in try/catch → logs an error and resolves, so a failed audit write never breaks the audited operation.

## 5. `@openldr/users`

```ts
export interface User {
  id: string; subject: string | null; username: string; displayName: string | null;
  email: string | null; roles: string[]; status: 'active' | 'disabled';
  lastLoginAt: string | null;
}
export interface UserStore {
  create(input: { username: string; displayName?: string; email?: string; roles?: string[] }): Promise<User>;
  get(id: string): Promise<User | undefined>;
  getBySubject(subject: string): Promise<User | undefined>;
  getByUsername(username: string): Promise<User | undefined>;
  list(): Promise<User[]>;
  setRoles(id: string, roles: string[]): Promise<void>;
  setStatus(id: string, status: 'active' | 'disabled'): Promise<void>;
  /** Just-in-time provision/link from verified token claims. */
  syncFromClaims(claims: TokenClaims): Promise<User>;
}
export function createUserStore(db: Kysely<InternalSchema>): UserStore;
```

`syncFromClaims` resolution order: **by `subject` → by `username` (link the subject onto the existing row) → create new** (subject set, status active); always bumps `last_login_at`. Username/email/name read from `preferred_username`/`email`/`name` claims with `sub` fallbacks. `create` throws on a duplicate username (unique constraint) — the CLI maps it to a clean error. Nothing calls `syncFromClaims` this round (login deferred); it is built and unit-tested for when auth wires in.

## 6. Integration — auditing what mutates (P1-AUD-2)

- **Ingest:** `handleIngestEvent` (`@openldr/ingest`) gains an optional `audit?(e: AuditEventInput): Promise<void>` dep (a callback type — keeps ingest adapter-free, no `@openldr/audit` import). `createIngestContext` injects `(e) => safeRecord(auditStore, logger, e)`. On `markDone`: record `{actorType:'system', actorName:'system', action:'ingest.batch.done', entityType:'batch', entityId:batchId, metadata:{source, converter, pluginId, pluginVersion, count}}`. On `markFailed`: `ingest.batch.failed` with `metadata.error`. This is the provenance↔audit link.
- **Plugins:** bootstrap wraps `ctx.plugins.install`/`remove` to `safeRecord` `plugin.install` / `plugin.remove` (entity=plugin, id=`<id>@<version>`, metadata={sha256}).
- **DB reset:** the CLI `db reset` path records `db.reset` (actor=system) after a successful reset.

## 7. Bootstrap & CLI

**Bootstrap:** `AppContext` gains `audit: AuditStore` and `users: UserStore` (built over `internal.db` in `createAppContext`); `createIngestContext` injects the ingest audit callback and wraps plugin install/remove.

**CLI** (`packages/cli/src/audit.ts` + `user.ts`, registered in `index.ts`):
- `audit list [--actor <id>] [--entity-type <t>] [--entity-id <id>] [--action <a>] [--from <iso>] [--to <iso>] [--json]`.
- `user list [--json]`, `user show <id> [--json]`, `user create --username <u> [--name <n>] [--email <e>] [--role <r...>] [--json]`, `user set-role <id> <roles...> [--json]`, `user activate <id>`, `user deactivate <id>`.

All emit `--json` (P1-CLI-2). `db reset` records an audit event (§6).

## 8. Error handling

- Operational audit writes are **best-effort** (`safeRecord` catches + logs; the audited op proceeds).
- Append-only is enforced by the store API (no update/delete methods).
- `user create` duplicate username → store throws (unique violation) → CLI prints a clean message + exit 1.
- `set-role`/`set-status`/`show` on an unknown id → no-op/`undefined` → CLI reports "user not found", exit 1.
- Empty/malformed `syncFromClaims` (missing `sub`) → throws a typed error (the future auth caller maps to 401-ish); unit-tested.

## 9. Testing & acceptance

**Unit (hermetic):**
- audit: `AuditStore.record` input→row mapping (and back to `AuditEvent`); `safeRecord` swallows a throwing store (logs, resolves); `list` filter→query mapping where extractable as a pure helper.
- users: `syncFromClaims` three branches (by-subject, by-username-link, create) + missing-`sub` error, against a fake store; create/setRoles/setStatus behavior with a fake store.
- CLI: `audit list` + `user *` handlers against a fake `ctx` (table vs `--json`, not-found exit codes).

**Integration (docker):**
- `db reset` → migrate (now includes `audit_events`, `users`). Ingest the WHONET sample → `openldr audit list --entity-type batch --json` shows an `ingest.batch.done` event with provenance metadata (P1-AUD-2). Install a plugin → an audit `plugin.install` row. `user create --username op --role admin` → `user list` shows it → `user set-role` updates. The `syncFromClaims` link is exercised by a `pnpm exec tsx -e` one-liner that builds the context and calls `ctx.users.syncFromClaims({ sub: 'kc-1', preferred_username: 'op' })`, then asserts the `op` row now has `subject='kc-1'` (no extra CLI surface added).

**Final gate:** `pnpm -r typecheck && pnpm -r test && pnpm depcruise && pnpm -r build` green; depcruise confirms `@openldr/audit` and `@openldr/users` import no `adapter-*`.

## 10. Done criteria (maps to PRD §5.7–5.8)

- [ ] Append-only `audit_events` + `AuditStore` (record/list/get) capturing actor/action/entity/before-after/timestamp (P1-AUD-1).
- [ ] Ingest batch + plugin lifecycle + db reset emit audit events; ingest events carry provenance metadata (P1-AUD-2).
- [ ] `users` table keyed by internal id + nullable IdP `subject`; `UserStore` with `syncFromClaims` JIT linking; decoupled from Keycloak (P1-USER-1).
- [ ] CLI `audit list` + `user list|show|create|set-role|activate|deactivate` with `--json` (P1-CLI-1/2).
- [ ] DP-1 intact (audit/users import no adapter); audit writes best-effort (never break the audited op).
- [ ] `pnpm -r typecheck && test && depcruise && build` green; live docker acceptance shows the ingest→audit link + user lifecycle.

## 11. Out of scope (deferred)

- Authentication / OIDC login flow and the actual `verifyToken`→`syncFromClaims` wiring (a later auth sub-project) — `syncFromClaims` is built but not invoked.
- Role-based authorization enforcement (roles are stored, not enforced).
- Users/Audit **UI** surfaces (dashboard nav stays disabled-stub).
- Audit log tamper-proofing / hash-chaining / retention policy.
- Audit of read/query operations (only mutations are audited).
