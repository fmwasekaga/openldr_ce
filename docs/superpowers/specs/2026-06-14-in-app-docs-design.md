# In-App Documentation (P2-DOC) — Design Spec

**Phase-2 §7 step 6.** Status: design approved, ready for implementation plan.

## Goal

Ship a searchable, i18n-capable, versioned in-app documentation system rendered inside the `apps/web` SPA, mirroring the Corlix bundled-markdown approach (reimplement, never copy — PRD §10). Author the full shipped-feature manual in English; provide the en/fr/pt mechanism with fr/pt falling back to English until translated. Regenerate doc screenshots via the existing `@openldr/e2e` Playwright harness.

## Requirements traceability

- **P2-DOC-1** — markdown rendered in the SPA, **searchable** (Fuse.js), **i18n (en/fr/pt)** mechanism with en authored + fr/pt fallback, **versioned with the product** (version-folder, latest fallback).
- **P2-DOC-2** — screenshots maintained alongside docs and regenerated via Playwright (extend `@openldr/e2e` capture).
- **P2-DOC-3** — setup guides including DHIS2 connection/mapping and external-DB configuration.

## Decisions (from brainstorming)

1. **i18n:** full mechanism + English content; fr/pt fall back to en until translated.
2. **Search:** add a search library — **Fuse.js** (fuzzy, ranked, small).
3. **Screenshots:** wire Playwright capture into the existing `@openldr/e2e` harness.
4. **Content set:** full shipped-feature set (~9 English pages).
5. **Architecture:** Approach A — bundled client-side docs inside `apps/web`; **no server route**, no new workspace package.

## Architecture

Docs are bundled into the SPA at build time. The browser holds the markdown (Vite `?raw`) and screenshot URLs (Vite `?url`); search runs client-side over the active locale. The server is unchanged — docs ship in `apps/web/dist` and are served by the existing `@fastify/static` SPA handler.

## Component design

### `version.ts`
`export const DOCS_VERSION = '0.1.0';` — the documentation version folder. Matches the `apps/web` package version; "versioned with product" is satisfied by the version-named folder plus `getDocsForVersion`'s latest fallback.

### `registry.ts`
- Glob: `import.meta.glob('./0.1.0/*/*.md', { query: '?raw', eager: true, import: 'default' })` → `Record<path, content>`. Parse each path `./0.1.0/<locale>/<slug>.md` into a nested map `BY_VERSION[version][locale][slug] = content`.
- `DOC_ORDER: string[]` — slugs in navigation order: `overview, getting-started, dashboard, reports, ingestion, terminology, dhis2, external-db, cli`.
- `export type Locale = 'en' | 'fr' | 'pt';` and `export const LOCALES: Locale[] = ['en', 'fr', 'pt'];`
- `firstHeading(md: string): string` — returns the text of the first `# H1`, trimmed; falls back to the slug (title-cased) when no H1.
- `resolve(locale: Locale, slug: string): DocSection | null` where `DocSection = { slug; title; content; localeUsed: Locale }`. Looks up `[version][locale][slug]`; **if absent, falls back to `[version]['en'][slug]`** (`localeUsed` records which was used). Returns `null` if even en is missing. `title = firstHeading(content)`.
- `list(locale: Locale): DocSection[]` — `DOC_ORDER` mapped through `resolve`, dropping nulls (defensive; en is always authored).
- `getDocsForVersion(version: string)` — returns the requested version's locale map, else the latest version key (sorted). One version exists now; this future-proofs multiple doc versions.

Version source for the active doc set is `DOCS_VERSION`.

### `search.ts`
- Record shape: `{ slug, title, headings: string, body: string }`. `headings` = all `##`/`###` text joined; `body` = markdown with formatting stripped to plain-ish text.
- `buildIndex(sections: DocSection[]): Fuse<DocRecord>` with keys weighted `title: 0.5, headings: 0.3, body: 0.2`, options `{ includeMatches: true, ignoreLocation: true, threshold: 0.3, minMatchCharLength: 2 }`.
- `searchDocs(index, query): SearchHit[]` where `SearchHit = { slug; title; snippet }`. Empty/whitespace query → `[]` (caller shows the full list). `snippet` = a ±60-char window around the first body match (or the title when the match is in the title), single-line, ellipsized.

### `screenshots.ts`
- `import.meta.glob('./0.1.0/screenshots/*.png', { eager: true, query: '?url', import: 'default' })` → map keyed by **basename** (`dashboard.png` → hashed URL). Screenshots are locale-independent (one set per version).
- `resolveImg(src: string): string | null` — returns the bundled URL for a bare basename, else `null` (caller omits the image and warns in dev). Absolute/`http(s)` srcs pass through unchanged.

### `DocMarkdown.tsx`
`<DocMarkdown content={md} />` wrapping `react-markdown` with `remarkPlugins={[remarkGfm]}` and component overrides:
- `img`: resolve `src` via `resolveImg`; render `<img loading="lazy" style={{maxWidth:'100%',border:'1px solid var(--border)',borderRadius:8}}>` or nothing if unresolved.
- `h1,h2,h3`: add `id` = slugified text (for anchors).
- `a`: external (`http`) links get `target="_blank" rel="noopener noreferrer"`; in-page `#` anchors unchanged.
- `code`/`pre`: plain monospace block styled with tokens — no syntax-highlighter dependency.

### `useDocLocale.ts`
Mirrors `useTheme`: `const KEY = 'openldr-docs-locale';` default `'en'`; `[locale, setLocale]` persisted to `localStorage` (guarded try/catch). No `data-*` attribute needed (docs-scoped, not global theming).

### Pages
- `Docs.tsx` (`/docs`): header with the language picker (`en/fr/pt` select) and a search input; when the query is empty, render the `list(locale)` page list (title + first-line excerpt) plus the `overview` page inline as a landing; when non-empty, render `searchDocs` hits (title + snippet) linking to `/docs/:slug`. Wrapped in `AppShell title="Documentation"`.
- `DocPage.tsx` (`/docs/:slug`): `resolve(locale, slug)`; render `<DocMarkdown>` inside `AppShell title={section.title}`. Unknown slug → the existing not-found card. Includes a "← All docs" back link and (when `localeUsed !== locale`) a subtle "Shown in English (not yet translated)" note.

### Routing + nav
- `App.tsx`: add `<Route path="/docs" element={<Docs />} />` and `<Route path="/docs/:slug" element={<DocPage />} />`.
- `AppShell.tsx`: add `{ to: '/docs', label: 'Docs', end: false, icon: IconDocs }` to `NAV`. `Forms/Users/Audit` SOON stubs remain.
- `icons.tsx`: add `IconDocs` (book/help glyph) using the shared `base(props)`.

## Screenshots / Playwright (P2-DOC-2)

Extend `@openldr/e2e`:
- New project `docs-capture` in `playwright.config.ts` (testDir `capture`, matching `docs-screenshots.spec.ts`), excluded from the default `smoke` run; invoked only by the new script.
- `capture/docs-screenshots.spec.ts`: a `DOC_SHOTS` manifest `[{ name, path, theme, width, height, fullPage, chart }]` covering at least `dashboard`, `report-amr`, `docs`, `doc-dhis2`. Reuses the proven capture helper pattern (theme seeded via `addInitScript` on `localStorage['openldr-theme']` + `data-theme`; animations killed; waits on `.recharts-surface` when `chart`). **Output dir = the committed `apps/web/src/docs/0.1.0/screenshots/`** (not `e2e/artifacts`, which stays gitignored).
- Root script: `"docs:screenshots": "playwright test --project=docs-capture"` (run after `e2e:install-browsers`, like `verify:ui`). Not part of `turbo test`.
- Markdown references shots by basename, e.g. `![Dashboard](dashboard.png)`.
- The committed PNGs let the bundle build without first running Playwright; regeneration is a maintenance step.

## Error handling

- Unknown slug → not-found card (existing pattern), HTTP-agnostic (client route).
- Missing locale file → en fallback (core requirement). Missing even en for a `DOC_ORDER` slug → `resolve` returns `null`; `list` drops it, `DocPage` shows not-found. (Authoring guarantees en exists for every ordered slug; a unit test asserts this.)
- Unresolved screenshot basename → image omitted + `console.warn` in dev; never a broken `<img>`.
- Empty search query → full page list (not an empty state).

## Testing strategy

**Unit (vitest, `apps/web`):**
- `registry.test.ts`: `resolve('en', slug)` returns content + H1 title; `resolve('fr', slug)` falls back to en with `localeUsed === 'en'`; unknown slug → null; `list('en')` returns `DOC_ORDER` length in order; every `DOC_ORDER` slug resolves in en (guards authoring completeness); `firstHeading` parses/falls back.
- `search.test.ts`: a query matching a title ranks that page first; a body-only term returns the page with a snippet containing the term; empty query → `[]`.
- `screenshots.test.ts`: known basename resolves to a (mocked) URL; unknown → null; `http` src passes through.

**Component (@testing-library/react):**
- `Docs.test.tsx`: renders the page list; typing a query narrows to matching titles; the language picker switching to `fr` still renders en content.
- `DocPage.test.tsx`: renders the resolved H1; an `img` with a known basename gets the bundled `src`; unknown slug shows not-found.
- `DocMarkdown.test.tsx`: gfm table/list render; external link gets `target=_blank`; screenshot img resolved.

**e2e smoke (extend `e2e/tests/smoke.spec.ts` or a new spec):** `/docs` shows the page list; typing in search narrows results; clicking a page renders its H1 and at least one `<img>`.

**Live acceptance:** `pnpm --filter @openldr/web build` → run built server from repo root → verify `/docs` renders, search works, language picker falls back to en; run `pnpm docs:screenshots` to regenerate PNGs and confirm they land in the committed folder and render in a doc page.

## Out of scope (YAGNI / deferred)

- App-wide UI i18n (only documentation is i18n per PRD).
- French/Portuguese translation content (mechanism only; fr/pt fall back to en).
- MDX / embedded live components (raw markdown via react-markdown; revisit if a page needs interactivity).
- Server-side docs API / search endpoint.
- Pixel-baseline screenshot diffing (capture only; agent/manual review as with `verify:ui`).
